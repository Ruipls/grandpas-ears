# Grandpa's Ears（爷爷的耳朵）—— 项目总陈述

## 一、项目背景与初心

### 1.1 我们为什么做这个

在中国，有超过 2700 万听障人士。他们中的许多人佩戴助听器或植入人工耳蜗，但在嘈杂环境、远距离对话、或与陌生人交流时，仍然面临巨大的沟通障碍。

**"听不见，但可以看。"** —— 这是 Grandpa's Ears 存在的最朴素理由。

我们希望通过技术，将真实世界中的语音实时转化为文字，让听障人士"看见"对方说的话，从而打破沟通的壁垒。

### 1.2 名字的由来

"Grandpa's Ears"（爷爷的耳朵）源于一个温暖的意象：如果爷爷年纪大了、听力下降，我们希望成为他的"耳朵"——帮他听清这个世界的声音，让他不再因为听不见而被隔绝在对话之外。

## 二、产品愿景

> **让每一次对话，都被看见。**

Grandpa's Ears 是一款面向听障人士的**实时语音转文字应用**。它的核心体验是：打开手机，放在桌上，对话双方的声音被实时识别并展示为文字——像一个"实时字幕"的对话界面。

## 三、目标用户

### 主要用户
- **听障人士**（佩戴助听器/人工耳蜗但仍需视觉辅助的用户）
- **老年性听力衰退者**
- **任何在特定场景下需要"看见声音"的人**

### 对话伙伴
- 家人、朋友、同事
- 医生、店员、服务人员
- 任何与听障人士交流的普通人

## 四、核心原则

1. **实时优先**：延迟必须足够低，不能让用户"等"字幕。
2. **简单即正义**：打开即用，不需要复杂的设置。目标用户中包含老年人，界面必须足够大、足够清晰。
3. **隐私至上**：语音数据默认不上传云端（端侧识别优先），或提供明确的隐私选项。
4. **离线可用**：核心功能在无网络环境下也能工作。
5. **无障碍设计**：从第一天起就遵循无障碍设计规范（大字体、高对比度、震动反馈等）。

## 五、分阶段规划

### 第一阶段：一对一对话（MVP）

**目标：** 实现两人面对面交流场景下的实时语音转文字。

**核心功能：**
- 实时语音识别（ASR），将说话内容转化为文字
- 对话流式界面：双方说话内容以聊天气泡形式展示，区分"我"和"对方"
- 基本的说话人区分（基于麦克风通道或手动标记）
- 对话历史记录与回看
- 大字体 / 高对比度显示模式

**不做什么（第一阶段）：**
- 不自动识别说话人身份
- 不支持多人（>2）对话
- 不做复杂的数据分析

### 第二阶段：声纹识别与智能增强

**目标：** 引入声纹识别，自动区分不同说话人，支持更复杂的对话场景。

**核心功能：**
- 声纹注册与识别：自动识别"这是谁在说话"
- 多人对话支持（>2 人）
- 说话人标签自动显示（如"妈妈"、"医生"）
- 环境噪音过滤增强
- 关键词高亮 / 重要信息提取

### 第三阶段（远期愿景）

- 翻译功能（多语言对话）
- 语气 / 情绪识别
- 可穿戴设备适配（智能眼镜等）
- 社区与无障碍生态共建

## 六、技术方向（初步）

| 层面 | 方案方向 |
|------|----------|
| **语音识别（ASR）** | Whisper / Web Speech API / 本地端侧模型 |
| **声纹识别** | Speaker Diarization / 声纹 embedding 模型 |
| **前端框架** | React / React Native（跨平台优先） |
| **后端（如需要）** | Node.js / Python（轻量优先） |
| **数据存储** | 本地优先（SQLite / IndexedDB），可选云端同步 |
| **目标平台** | iOS / Android（移动端优先），后续考虑 Web 版 |

## 七、成功指标

### 第一阶段 MVP 成功标准：
- 语音识别延迟 < 500ms（从说话结束到文字显示）
- 中文普通话识别准确率 ≥ 95%
- 应用冷启动到开始识别 < 3 秒
- 用户无需阅读说明书即可完成第一次对话

### 长期目标：
- 月活跃用户 10,000+
- App Store / 应用商店评分 ≥ 4.5
- 真实用户反馈："它真的帮到了我"

## 八、非功能需求

- **无障碍（A11y）**：遵循 WCAG 2.1 AA 标准，支持系统辅助功能（TalkBack / VoiceOver）
- **国际化（i18n）**：首发中文，架构预留多语言扩展能力
- **性能**：在入门级手机上也能流畅运行
- **电量**：持续识别场景下，每小时耗电 ≤ 15%
